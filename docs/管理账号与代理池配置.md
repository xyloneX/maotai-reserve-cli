# 管理账号与代理池配置

## 一、两个 Web 登录账号（已内置）

| 用途 | 用户名 | 角色 | 能做什么 |
|------|--------|------|----------|
| **你（最高权限）** | `owner` | superadmin | 全部功能，含 **系统设置、代理池、改 config** |
| **甲方（日常使用）** | `client` | operator | 账号/批量登录/预约/中签/任务/记录，**不能改系统设置和代理池** |

密码写在服务器 `/opt/maotai/deploy/.env`（部署时生成，勿提交 Git）：

```env
MT_OWNER_USERNAME=owner
MT_OWNER_PASSWORD=你的强密码
MT_CLIENT_USERNAME=client
MT_CLIENT_PASSWORD=甲方专用密码
```

改密码后执行：`sudo systemctl restart maotai-api`

旧单账号 `admin` 仍可作为 owner 的备用（`MT_ADMIN_USERNAME` / `MT_ADMIN_PASSWORD` 与 owner 同步逻辑兼容）。

## 二、代理池怎么配（1000 号必读）

### 原则

- 每个账号在 DB 里有一个 **`egress_group`**（出口组名，如 `ip001`）
- `config.yaml` 里 **`proxy_pools`** 把组名映射到代理 URL
- **同一组内**账号串行预约（共享一个 IP，避免打爆）
- **不同组**并行预约（已实现 `reserve.parallel_by_egress`）

### 推荐规模

| 账号数 | 建议 |
|--------|------|
| 1～20 | 可不配代理，直连即可 |
| 20～100 | 5～10 个代理，每组 10～20 号 |
| 100～1000 | **50 个代理**，每组 **20 号**（`assign_egress_groups.py --per-group 20`） |

### 操作步骤（Web）

1. 用 **`owner`** 登录 → **系统设置** → **代理池**
2. 点击 **「从账号同步出口组」**：按已导入账号的 `egress_group` 生成空行
3. 在每行 **代理 URL** 填入真实地址，例如：
   - `http://user:pass@1.2.3.4:8080`
   - `socks5://user:pass@5.6.7.8:1080`
4. **检测全部代理** → 确认绿色可用
5. **保存代理配置**（写入 `config.yaml`）

### 操作步骤（命令行批量分组）

```bash
# 1. 给 credentials / DB 账号批量打组名 ip001, ip002 ...
python scripts/assign_egress_groups.py --per-group 20

# 2. 编辑 config.yaml 补全 proxy_pools（或用 Web 保存）

# 3. 同步到服务器 DB
# Web 账号管理 → 同步凭证文件，或 API sync-credentials
```

### config.yaml 示例（50 组 × 20 号 = 1000 号）

```yaml
proxy_pools:
  ip001: "http://代理商提供的地址1"
  ip002: "http://代理商提供的地址2"
  # … 共 50 项

max_accounts_per_egress: 20
egress_group_stagger_seconds: 3

reserve:
  parallel_by_egress: true
  max_workers: 32
  shard_size: 50
```

### 没有代理时

- 可留空 `proxy_pools: {}`
- 所有账号 `egress_group` 留空 → 走服务器本机 IP（仅适合少量号）

### 代理从哪里来

- 代理服务商（HTTP/SOCKS5 住宅/4G 代理）
- 多台手机热点（每组连不同热点，在组内用该热点出口，可不填 URL、分机部署）
- 多条宽带 / 云手机不同出口

## 三、甲方账号使用说明（可转发）

1. 登录地址：http://你的服务器IP/
2. 用户名：`client`，密码由你单独提供
3. 常用：**批量登录** → 发码 → 填验证码；**任务中心** → 预约
4. 无法修改系统参数和代理（需你方 `owner` 账号操作）
